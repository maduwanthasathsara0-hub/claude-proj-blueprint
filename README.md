# 🧩 claude-proj-blueprint - Build projects with clear structure

[![Download the latest release](https://img.shields.io/badge/Download-Release%20Page-blue?style=for-the-badge&logo=github)](https://github.com/maduwanthasathsara0-hub/claude-proj-blueprint/releases)

## 🖥️ What this is

claude-proj-blueprint is a project template for people who want a clear way to plan and build software with Claude Code, Obsidian, and Figma.

It gives you a level-based setup so you can pick the right starting point for your work:

- **L1** for a simple start
- **L2** for a more organized setup
- **L3** for deeper planning and team work
- **L4** for a full project structure

Use it to keep specs, notes, and design work in one place.

## 📦 What you get

This template is set up to help you:

- organize project notes
- keep requirements easy to find
- connect planning work with design work
- track progress in a simple folder layout
- work with Claude Code in a structured way
- store project knowledge in Obsidian
- keep design references close to the work
- use vector search tools like Chroma or pgvector for project memory

## 🧭 Pick your maturity level

Choose the level that fits your project:

### L1 - Start simple
Best for small ideas and early drafts.

Use this level when you want:

- a basic folder layout
- a simple spec file
- a place for notes and rough plans

### L2 - Stay organized
Best for solo work that needs more structure.

Use this level when you want:

- clearer task lists
- better project notes
- a cleaner handoff between planning and building

### L3 - Work across tools
Best for work that uses notes, code, and design together.

Use this level when you want:

- Obsidian notes linked to project work
- Figma references in the flow
- more detail around scope and steps

### L4 - Full project system
Best for larger projects with more moving parts.

Use this level when you want:

- a deeper spec tree
- better project tracking
- stronger structure for long work cycles
- room for vector search and project memory

## 🗂️ Typical folder layout

A common setup may include:

- `specs/` for project specs
- `notes/` for planning and research
- `design/` for Figma links and image refs
- `tasks/` for work items
- `docs/` for project documents
- `memory/` for reusable project context

This layout keeps your work in clear places so you can find it fast.

## ❓ Common questions

### Do I need coding knowledge?
No. You can use this as a structured folder and note system first. If you work with Claude Code or a developer later, the structure still helps.

### Can I use it for small projects?
Yes. Start with L1 and keep the setup light.

### Can I use it for team work?
Yes. L2 through L4 give you more room for shared planning and better project tracking.

### Does it work with design files?
Yes. It is made to fit design links and notes from Figma.

### Can I use it with project memory tools?
Yes. The structure works well with Chroma, pgvector, and similar tools.
